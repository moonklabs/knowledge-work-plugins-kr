# 일일 영업 브리핑

오늘 가장 중요한 것을 명확하게 파악합니다. 이 스킬은 알려주는 정보로 작동하며, 도구를 연결하면 더 풍부해집니다.

## 작동 방식

```
┌─────────────────────────────────────────────────────────────────┐
│                      DAILY BRIEFING                              │
├─────────────────────────────────────────────────────────────────┤
│  기본 기능 (단독 작동)                                            │
│  ✓ 사용자 제공: 오늘의 미팅, 주요 거래, 우선순위                    │
│  ✓ 정리: 우선순위별 하루 실행 계획                                 │
│  ✓ 출력: 2분 만에 읽을 수 있는 스캔 가능한 브리핑                   │
├─────────────────────────────────────────────────────────────────┤
│  강화 모드 (도구 연결 시)                                         │
│  + 캘린더: 오늘의 미팅 및 참석자 자동 가져오기                      │
│  + CRM: 파이프라인 알림, 작업, 거래 건강 상태                      │
│  + 이메일: 주요 어카운트 미읽음 메일, 답장 대기 중인 메일            │
│  + 데이터 보강: 어카운트의 야간 신호                               │
└─────────────────────────────────────────────────────────────────┘
```

---

## 시작하기

이 스킬 실행 시 필요한 정보를 요청합니다:

**캘린더 미연결 시:**
> "오늘 어떤 미팅이 있나요? (캘린더를 붙여넣거나 나열해 주세요)"

**CRM 미연결 시:**
> "이번 주 집중하고 있는 거래는? 주의가 필요한 건이 있나요?"

**커넥터가 있는 경우:**
모든 정보를 자동으로 가져와 브리핑만 보여줍니다.

---

## 커넥터 (선택사항)

도구를 연결하여 이 스킬을 강화하세요:

| 커넥터 | 추가 기능 |
|--------|----------|
| **캘린더** | 오늘의 미팅, 참석자, 시간 및 맥락 |
| **CRM** | 진행 중인 파이프라인, 마감 임박 거래, 지연 작업, 정체 거래 |
| **이메일** | 영업 기회 담당자의 미읽음 메일, 답장 대기 중인 메일 |
| **데이터 보강** | 야간 신호: 어카운트의 투자, 채용, 뉴스 |

> **커넥터가 없나요?** 문제없습니다. 미팅과 거래를 알려주시면 브리핑을 만들어 드립니다.

## 실행 흐름

### 1단계: 맥락 수집

**커넥터 사용 가능 시:**
```
1. 캘린더 → 오늘의 일정 가져오기
   - 외부 미팅 필터 (비사내 참석자)
   - 가져오기: 시간, 제목, 참석자, 설명

2. CRM → 파이프라인 조회
   - 본인 담당 진행 중인 영업 기회
   - 플래그: 이번 주 마감, 7일 이상 무활동, 일정 밀림
   - 가져오기: 지연 작업, 예정 작업

3. 이메일 → 우선 메시지 확인
   - 영업 기회 담당자 도메인 미읽음 메일
   - 답장 없는 발송 메시지 (3일 이상)

4. 데이터 보강 → 신호 확인 (사용 가능 시)
   - 진행 중 어카운트의 투자, 채용, 뉴스
```

**커넥터 미연결 시:**
```
사용자에게 질문:
1. "오늘 어떤 미팅이 있나요?"
2. "집중하고 있는 거래는? 마감 임박이거나 주의 필요한 건이 있나요?"
3. "긴급하게 알아야 할 사항이 있나요?"

제공되는 정보로 작업 진행.
```

### 2단계: 우선순위 결정

```
우선순위 순서:
1. 긴급: 오늘/내일 마감인데 아직 미성사 거래
2. 높음: 오늘 고가 영업 기회 미팅
3. 높음: 의사결정자의 미읽음 이메일
4. 중간: 이번 주 마감 거래
5. 중간: 정체 거래 (7일 이상 무활동)
6. 낮음: 이번 주 기한 작업

#1 우선순위 선택:
- 오늘 >$50K 거래 미팅 → 준비 집중
- 오늘 마감 거래 → 성사에 집중
- 바이어의 긴급 이메일 → 우선 답장
- 그 외 → 최고가 정체 거래
```

### 3단계: 브리핑 생성

```
사용 가능한 데이터 기반 섹션 구성:

1. #1 우선순위 — 항상 포함 (단순해도)
2. 오늘의 수치 — CRM 연결 시, 아니면 생략
3. 오늘의 미팅 — 캘린더 또는 사용자 입력에서
4. 파이프라인 알림 — CRM 연결 시
5. 이메일 우선순위 — 이메일 연결 시
6. 권장 조치 — 항상 상위 3개 조치 포함
```

## 빠른 모드

"quick brief" 또는 "tldr my day"라고 말하면 축약 버전을 제공합니다:

```markdown
# 빠른 브리핑 | [날짜]

**#1:** [우선 조치]

**미팅:** [N]건 — [기업 1], [기업 2], [기업 3]

**알림:**
- [알림 1]
- [알림 2]

**지금 할 일:** [가장 중요한 단일 조치]
```

## 하루 마무리 모드

마지막 미팅 후 "wrap up my day" 또는 "end of day summary"라고 말하세요:

```markdown
# 하루 마무리 | [날짜]

**완료:**
- [미팅 1] — [결과]
- [미팅 2] — [결과]

**파이프라인 변동:**
- [거래]가 [단계]로 이동

**내일의 초점:**
- [우선순위 1]
- [우선순위 2]

**미결 사항:**
- [ ] [후속 조치가 필요한 미완료 항목]
```

## 팁

1. **캘린더를 먼저 연결** — 가장 큰 시간 절약
2. **CRM을 두 번째로 추가** — 파이프라인 알림 활성화
3. **커넥터 없이도** — 미팅을 알려주시면 우선순위 정리를 도와드립니다
