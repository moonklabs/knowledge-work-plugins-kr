# Write Spec

> 낯선 플레이스홀더가 보이거나 어떤 도구가 연결되어 있는지 확인하려면 [CONNECTORS.md](../CONNECTORS.md)를 보세요.

문제 정의에서 실행 가능한 PRD를 생성합니다.

## Workflow

1. 문제/목표/대상 사용자 정리
2. 성공 지표와 비목표 정의
3. 요구사항(`must/should/could`) 구조화
4. 사용자 스토리/수용 기준 작성
5. 리스크, 오픈 이슈, 롤아웃 계획 명시

## Output

- PRD draft
- Requirements table
- Acceptance criteria
- Open questions
