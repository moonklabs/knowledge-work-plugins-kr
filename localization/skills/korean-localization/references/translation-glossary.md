# Translation Glossary

Korean terminology reference for translating Claude Code plugins across 11 domains. Use these translations consistently across all plugin files.

## How to Use This Glossary

- **First use**: Include English in parentheses for clarity: "워크플로우(workflow)"
- **Subsequent uses**: Korean term only: "워크플로우"
- **Technical context**: Keep English if it's a product name, tool name, or widely-used tech term
- **Consistency**: Always use the same Korean term for the same English concept

## Usage Notes

### When to Keep English

Keep English terms in these cases:
- **Brand names**: Slack, Linear, Jira, GitHub, Notion, etc.
- **Product names**: Claude, Anthropic, PubMed, etc.
- **Widely-used acronyms**: API, CLI, SDK, JSON, YAML, SQL (database), etc.
- **Tool names**: Read, Write, Grep, Bash (Claude's tools)
- **Tech stack**: React, Python, TypeScript, etc.

### When to Use Parenthetical English

On **first use** of technical terms, include English for clarity:
- "워크플로우(workflow)를 정의합니다"
- "백로그(backlog)를 관리합니다"

On **subsequent uses**, use Korean only:
- "워크플로우를 실행합니다"
- "백로그를 업데이트합니다"

### Consistency is Key

Always use the **same Korean term** for the same English concept within a plugin and across the entire repository. This glossary ensures consistency.
