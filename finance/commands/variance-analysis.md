# Variance / Flux Analysis

> 낯선 플레이스홀더가 보이거나 어떤 도구가 연결되어 있는지 확인하려면 [CONNECTORS.md](../CONNECTORS.md)를 보세요.

**중요**: 이 커맨드는 변동 분석을 보조하지만 재무 자문을 제공하지 않습니다. 보고 전 전문가 검토가 필요합니다.

## Usage

```bash
/flux <area> <period-comparison>
```

### Arguments

- `area`: `revenue`, `opex`, `capex`, `headcount`, `cogs`, `gross-margin` 또는 계정 그룹
- `period-comparison`: `2024-12 vs 2024-11`, `2024-Q4 vs 2023-Q4`, `2024-12 vs budget` 등

## Workflow

### 1. 데이터 수집

연결 소스가 있으면 실제값/비교값/운영지표를 수집합니다.
연결이 없으면 사용자에게 두 기간 데이터와 드라이버 지표(물량, 단가, 인원 등)를 요청합니다.

### 2. 상위 변동 계산

총액 기준 `Variance($)`와 `Variance(%)`를 계산합니다.

### 3. 드라이버 분해

영역별 표준 분해를 적용:
- Revenue: 물량/단가/믹스/신규-기존/환율
- OpEx: 인원/보상/볼륨연동/신규투자/일회성/타이밍
- CapEx: 프로젝트/일정/범위/원가
- Headcount: 채용속도/이탈/보상믹스/계약직

### 4. 워터폴 제시

텍스트 워터폴로 각 드라이버 기여도를 연결해 총변동과 일치시키고, `Unexplained` 잔차를 표시합니다.

### 5. 내러티브 작성

중요 드라이버마다:
- 유리/불리
- 금액/비율
- 2-3문장 원인 설명
- 향후 전망(지속/반전/일회성)

### 6. 미설명 변동 플래그

잔차가 크면 조사 후보 영역과 추가 질문을 제시합니다.

### 7. 출력

1. 상위 변동 요약
2. 드라이버 분해 표
3. 워터폴
4. 중요 드라이버 내러티브
5. 미설명 변동 및 조사 제안
6. 최근 추세 맥락
7. 후속 액션
