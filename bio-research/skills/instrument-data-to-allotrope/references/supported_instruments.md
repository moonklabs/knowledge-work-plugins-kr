# 지원 기기

## 이 스킬로 무엇을 변환할 수 있습니까?

**Allotrope 스키마에 매핑되는 모든 기기 데이터를 변환할 수 있습니다.** 이 스킬은 계층화된 파싱 방식을 사용합니다:

1. **네이티브 allotropy 파서** (아래 나열) — 벤더별 형식에 대해 검증된 최고 충실도
2. **유연한 폴백 파서** — 열을 ASM 필드에 매핑하여 모든 테이블 형식 데이터(CSV, Excel, TXT) 처리
3. **PDF 추출** — PDF에서 테이블을 추출한 후 유연한 파싱 적용

아래에 기기가 나열되어 있지 않더라도, 데이터에 ASM 기술 스키마에 매핑되는 인식 가능한 측정 필드(시료 ID, 값, 단위, 타임스탬프 등)가 포함되어 있으면 변환할 수 있습니다.

## 자동 감지 패턴

이 스킬은 다음 패턴을 사용하여 파일 내용에서 기기 유형을 식별합니다:

### Vi-CELL BLU
- 열 헤더: "Sample ID", "Viable cells (x10^6 cells/mL)", "Viability (%)"
- 파일 구조: 특정 열 순서를 가진 CSV

### Vi-CELL XR
- 열 헤더: "Sample", "Total cells/ml", "Viable cells/ml"
- 다양한 내보내기 형식 지원

### NanoDrop
- 열 헤더: "Sample Name", "Nucleic Acid Conc.", "A260", "A280"
- 260/280 및 260/230 비율 열

### 플레이트 리더 (일반)
- 웰 식별자 (A1-H12 패턴)
- "Plate", "Well", "Sample" 열
- 메타데이터 헤더가 있는 블록 기반 구조

### ELISA
- 농도가 있는 표준 곡선 데이터
- OD/흡광도 판독값
- 시료/블랭크/표준물질 분류

## Vendor Enum 사용

```python
from allotropy.parser_factory import Vendor
from allotropy.to_allotrope import allotrope_from_file

# List all supported vendors
for v in Vendor:
    print(f"{v.name}: {v.value}")

# Convert file
asm = allotrope_from_file("data.csv", Vendor.BECKMAN_VI_CELL_BLU)
```

## 지원 상태 확인

```python
from allotropy.parser_factory import get_parser

# Check if a vendor/file combo is supported
try:
    parser = get_parser(Vendor.BECKMAN_VI_CELL_BLU)
    print("Supported!")
except Exception as e:
    print(f"Not supported: {e}")
```
