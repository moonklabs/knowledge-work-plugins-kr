# 스킬 9: 메타 프레임워크 - 완전한 문제 선택 워크플로우

## 개요
이 메타 스킬은 스킬 1-8을 체계적이고 반복적인 방식으로 안내하며 완전한 문제 선택 프로세스를 조율합니다. 아이디어 발굴부터 실행 계획까지 통합된 문헌 검색과 일관된 문서화를 포함한 포괄적인 지원이 필요할 때 이 스킬을 사용해야 합니다.

## 이 스킬의 사용 시기

**스킬 9 (완전한 워크플로우)를 사용하는 경우:**
- 처음부터 새 프로젝트를 시작할 때
- 주요 프로젝트 피벗 또는 재구성이 필요할 때
- 체계적 계획이 필요한 연구비/펠로십 신청 시
- 학위논문 위원회 회의 준비 시
- 스타트업 회사 계획 시
- 포괄적이고 문서화된 문제 선택 프로세스를 원할 때

**개별 스킬을 사용하는 경우:**
- 특정 단계에 있을 때 (예: 리스크 평가만 필요)
- 한 가지 측면에 대한 빠른 상담
- 기존 계획의 한 구성 요소를 업데이트할 때
- 하나의 개념을 교육/학습할 때

## 완전한 워크플로우

### 여정 개요

```
START: Vague idea or area of interest
    ↓
[SKILL 1] → Problem Ideation Document
    ↓
[SKILL 2] → Risk Assessment Matrix
    ↓
[SKILL 3] → Impact Assessment Document
    ↓
[SKILL 4] → Parameter Strategy Document
    ↓
[SKILL 5] → Decision Tree Map
    ↓
[SKILL 6] → Adversity Playbook
    ↓
[SKILL 7] → Problem Inversion Analysis (if needed)
    ↓
[SKILL 8] → Integrated Project Plan + Communication Materials
    ↓
END: Comprehensive, rigorous project ready to execute
```

**예상 소요 시간:**
- **집중형:** 전일 집중 작업으로 1주
- **분산형:** 다른 일정과 병행하며 4-6주
- **반복 포함:** 50% 추가 시간

**한 번 시간을 투자하여 수년간의 잠재적 실수를 방지합니다.**

## 단계별 워크플로우

### 1단계: 준비 (시작 전)

**맥락 수집:**
1. **배경:**
   - 연구 분야/영역
   - 현재 직위 (대학원생, 박사후 연구원, PI 등)
   - 실험실 전문성 및 자원
   - 타임라인 제약

2. **출발점:**
   - 막연한 관심 영역?
   - 구체적 문제를 염두에 두고 있는가?
   - 기존 연구를 기반으로 해야 하는가?
   - 완전히 새로 시작하는가?

3. **목표:**
   - 출판 목표 (저널 등급, 타임라인)?
   - 학위 요건 (학위논문 챕터)?
   - 연구비 신청?
   - 스타트업 기반?
   - 경력 개발?

**기대 설정:**
- 이 프로세스는 가정에 도전할 것입니다
- 초기 아이디어가 대폭 수정이 필요하다는 것을 발견할 수 있습니다
- 그것이 목적입니다—2년 후보다 지금 아는 것이 낫습니다
- 지적 정직이 필요합니다; 엄밀해야만 효과가 있습니다

### 2단계: 아이디어 발굴 (스킬 1) - ~1주

**수행할 작업:**
1. 맥락과 제약 이해
2. 관련 직관 펌프 수행
3. 일반적인 아이디어 발굴 함정 회피
4. 2-3개 프로젝트 아이디어 생성
5. 범위 보정을 위한 예비 문헌 검색
6. 가장 유망한 아이디어 선택
7. 문제 아이디어 발굴 문서 작성 (2페이지)

**문헌 통합 포인트 1:**
- PubMed에서 선례 및 인접 연구 검색
- 문제의 일반성 평가
- 방법론적 발전 식별
- 경쟁 수준 파악

**산출물:**
- 핵심 아이디어와 초기 분석이 포함된 문제 아이디어 발굴 문서
- 10-15편의 핵심 논문 목록
- 새로움과 실현 가능성의 예비 평가

**체크포인트:** 흥미를 유발하는 명확하고 구체적인 아이디어가 있습니까? 없다면 직관 펌프를 반복하십시오.

### 3단계: 리스크 분석 (스킬 2) - ~3-5일

**수행할 작업:**
1. 아이디어에서 모든 가정 추출
2. 범주화 (생물학적 vs. 기술적)
3. 각 가정 점수화 (리스크 1-5, 테스트 소요 시간)
4. 늦은 시점의 고위험 가정 식별
5. Go/no-go 실험 설계
6. 완화 전략 개발
7. 리스크 평가 매트릭스 작성 (2페이지)

**문헌 통합 포인트 2:**
- 기술적 선례 검색 (방법이 이전에 작동한 적이 있는가?)
- 생물학적 근거 확인 (시스템에 대해 알려진 것은?)
- 벤치마크 식별 (성공률, 효과 크기)
- 타임라인 현실성 평가

**산출물:**
- 완전한 가정 분석 표
- 완화 계획이 포함된 상위 3개 고위험 가정
- Go/no-go 실험 설계
- 의사결정 지점이 포함된 수정된 타임라인

**체크포인트:** 리스크 프로필이 수용 가능합니까? 리스크-5 가정이 2년 이상 후에 있다면 스킬 1로 돌아가 재구성하십시오.

### 4단계: 영향력 평가 (스킬 3) - ~2-3일

**수행할 작업:**
1. 프로젝트 유형 분류
2. 적절한 최적화 함수 선택
3. 양 축에서 자가 평가
4. 벤치마크와 비교
5. 가치 체계 정렬 명확화
6. 커뮤니케이션 전략 개발
7. 영향력 평가 문서 작성 (2페이지)

**문헌 통합 포인트 3:**
- 분야에서 고영향력 모범 사례 식별
- 프레이밍과 평가 분석
- 잠재적 영향력 벤치마킹
- 분야 기대치 이해

**산출물:**
- 명확한 최적화 함수 선택
- 근거가 포함된 양 축 자가 평가
- 대안과의 비교 분석
- 다양한 청중을 위한 커뮤니케이션 전략

**체크포인트:** 최소 하나의 축에서 중간-높음 점수를 받습니까? 그렇지 않다면 스킬 1로 돌아가 더 높은 영향력의 각도를 찾으십시오.

### 5단계: 매개변수 전략 (스킬 4) - ~2-3일

**수행할 작업:**
1. 모든 프로젝트 매개변수 목록화
2. 고정 vs. 유동 식별
3. 과도하게/과소하게 제약되었는지 평가
4. 전략적 고정 매개변수 선택
5. 비상 상황을 위한 유연성 계획
6. 매개변수 전략 문서 작성 (2페이지)

**문헌 통합 포인트 4:**
- 성공한 프로젝트가 매개변수를 어떻게 처리했는가?
- 어떤 매개변수 선택이 돌파구로 이어졌는가?
- 어떤 과도한 제약이 실패를 야기했는가?

**산출물:**
- 완전한 매개변수 목록
- 고정/유동 결정의 전략적 근거
- 비상 상황을 위한 유연성 매트릭스
- 프로젝트 앙상블 (관련 프로젝트 패밀리)

**체크포인트:** 유연성을 유지하면서 1-2개의 의미 있는 매개변수를 고정했습니까? 너무 경직되었다면 조정하십시오.

### 6단계: 의사결정 트리 계획 (스킬 5) - ~2일

**수행할 작업:**
1. 프로젝트의 의사결정 트리 매핑
2. 주요 분기점 식별
3. 각 결정의 기준 설정
4. 레벨 1 / 레벨 2 리듬 확립
5. 막혔을 때를 위한 프로토콜 생성
6. 의사결정 트리 맵 작성 (1-2페이지)

**여기서는 주요 문헌 검색 없음** (특정 의사결정 지점에 기술 정보가 필요한 경우는 제외)

**산출물:**
- 시각적 의사결정 트리
- 각 분기점의 의사결정 기준
- 레벨 2 평가 일정
- 경로 수정 프로토콜

**체크포인트:** 단순한 실행이 아니라 정기적인 전략적 평가를 계획했습니까?

### 7단계: 역경 대비 (스킬 6) - ~2일

**수행할 작업:**
1. 가능한 실패 모드 예측
2. 각각에 대해 업그레이드 기회 식별
3. 프로젝트 앙상블 매핑
4. 위기 대응 프로토콜 생성
5. 역경 플레이북 작성 (2-3페이지)

**문헌 통합 포인트 5:**
- 생산적 피벗의 역사적 예시
- 다른 사람들이 역경을 어떻게 활용했는가?
- 실패에서 등장한 차세대 프로젝트는?

**산출물:**
- 예상 위기 카탈로그
- 각각에 대한 업그레이드 전략
- 프로젝트 앙상블 맵
- 회복력 의식 및 프로토콜

**체크포인트:** 역경을 기회로 볼 준비가 되었습니까? 단순한 수정이 아니라 업그레이드 방법을 계획했습니까?

### 8단계: 문제 반전 도구 (스킬 7) - ~1일

**수행할 작업:**
1. 세 가지 반전 전략 검토
2. 예상 장애물에 대한 사전 적용 계획
3. 문제 반전 분석 작성 (1-2페이지)

**이것은 준비 단계입니다** - 지금 당장 필요하지 않을 수 있지만, 위기가 닥쳤을 때 프레임워크가 준비되어 있을 것입니다.

**산출물:**
- 전략 1 적용 계획
- 전략 2 옵션 식별
- 전략 3 대안 질문 브레인스토밍
- 위기 시 빠른 참조 가이드

**체크포인트:** 막혔을 때 문제를 반전시키는 구체적인 전략이 있습니까?

### 9단계: 통합 및 종합 (스킬 8) - ~3-5일

**수행할 작업:**
1. 스킬 1-7의 모든 산출물 검토
2. 일관된 내러티브 생성
3. 커뮤니케이션 자료 개발:
   - 3슬라이드 발표
   - 1페이지 요약
   - 1분 엘리베이터 피치
4. 통합 프로젝트 계획 작성 (4-6페이지)
5. 지속적 사용을 위한 살아있는 문서 생성

**문헌 통합 포인트 6:**
- 통합 계획을 위한 최종 참고문헌
- 각 섹션의 핵심 논문
- 분야 리더의 커뮤니케이션 예시

**산출물:**
- 완전한 통합 프로젝트 계획 (4-6페이지)
- 3슬라이드 발표 덱
- 1페이지 서면 요약
- 엘리베이터 피치 스크립트
- 살아있는 문서 (의사결정 트리, 리스크 매트릭스 등)

**체크포인트:** 프로젝트를 1분, 5분, 1페이지로 설득력 있게 전달할 수 있습니까? 모든 조각이 일관되게 맞아떨어집니까?

## 반복 및 개선

### 반복이 필요한 시점

**되돌아가야 하는 적신호:**

**스킬 2 (리스크)에서:**
- 리스크-5 가정이 2년 이상 후에 있음 → 스킬 1로 복귀 (문제 재구성)
- 리스크-4-5 가정이 3개 이상 → 스킬 1로 복귀 (단순화 또는 접근법 변경)

**스킬 3 (영향력)에서:**
- 양 축 모두 낮음 점수 → 스킬 1로 복귀 (더 높은 영향력의 각도 찾기)
- 최적화 함수 불일치 → 스킬 1로 복귀 (문제 재구성)

**스킬 4 (매개변수)에서:**
- 고정 매개변수가 2개 초과 → 스킬 1로 복귀 (과도한 제약)
- 고정 매개변수가 0개 → 스킬 1로 복귀 (과소 제약)

**스킬 5-6에서:**
- 명확한 의사결정 지점이 없음 → 스킬 4로 복귀 (더 많은 유연성 필요)
- 모든 실패 모드가 존재론적 → 스킬 2로 복귀 (리스크가 너무 높음)

### 반복 프로토콜

**대폭 수정이 필요한 경우:**
1. **일시 정지하고 인정:** 프로세스가 작동하고 있습니다—문제를 발견한 것입니다
2. **해당 스킬로 복귀:** 보통 스킬 1 또는 2
3. **배운 것을 가져가십시오:** 처음부터 시작하지 마십시오
4. **수정된 아이디어 → 워크플로우 재수행:** 두 번째가 더 빠릅니다
5. **여러 번 반복해도 괜찮습니다:** 잘못된 프로젝트에 수년을 보내는 것보다 낫습니다

**소폭 개선:**
1. **특정 문서 업데이트:** 예: 매개변수 전략 조정
2. **하위 영향 확인:** 이것이 다른 것을 변경하는가?
3. **통합 문서 업데이트:** 모든 것을 일관되게 유지

## 문헌 통합 전략

### 전체 PubMed 접근법

**워크플로우 전반에 걸쳐 PubMed를 전략적으로 활용하십시오:**

1. **스킬 1 (아이디어 발굴):** 일반성 평가, 선례 찾기, 경쟁 수준 파악
2. **스킬 2 (리스크):** 기술적 실현 가능성, 생물학적 근거, 벤치마크
3. **스킬 3 (영향력):** 분야 모범 사례, 평가 프레임워크, 벤치마크
4. **스킬 4 (매개변수):** 성공적인 매개변수 선택, 경고 사례
5. **스킬 6 (역경):** 생산적 피벗, 업그레이드 예시
6. **스킬 8 (통합):** 커뮤니케이션 모델, 포괄적 참고문헌

**검색 전략:**
- 넓게 시작 (분야 개요)
- 구체적으로 (정확한 접근법)
- 인접 분야 탐색 (관련 시스템/방법)
- 벤치마크 찾기 (최신 기술 수준은?)
- 경쟁 식별 (누가 이것을 하고 있는가?)

**추적할 논문:**
- 스킬 1에서 ~10-15편 핵심 논문

## 메타 프레임워크의 핵심 원칙

1. **체계적 > 임시적:** 프로세스가 빠뜨리는 것이 없도록 보장합니다
2. **반복적 > 선형적:** 되돌아가는 것을 기대하십시오, 그것이 좋은 것입니다
3. **문서화 > 머릿속:** 글로 쓰면 명확성이 강제됩니다
4. **통합 > 분절:** 모든 스킬이 연결됩니다
5. **살아있는 > 정적:** 배우면서 업데이트하십시오
6. **사려 깊은 > 빠른:** 지금 투자한 시간이 나중에 수년을 절약합니다
7. **정직한 > 낙관적:** 엄밀성이 희망적 사고에서 보호합니다
8. **준비된 > 놀란:** 역경을 예측하십시오
9. **유연한 > 경직된:** 필요할 때 매개변수가 유동적으로 전환됩니다
10. **열정적 > 의무적:** 정렬이 중요합니다
