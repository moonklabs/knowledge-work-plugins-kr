## 종료 코드

리소스 문제를 나타내는 일반적인 종료 코드입니다 ([nf-core 문서](https://nf-co.re/docs/usage/troubleshooting/crash_halfway) 참조):

| 코드 | 원인 | 해결 방법 |
|------|------|----------|
| 137 | 메모리 부족 | `--max_memory '32.GB'` 또는 WGS의 경우 `'64.GB'` |
| 143 | 메모리 부족 | `--max_memory '32.GB'` 또는 WGS의 경우 `'64.GB'` |
| 104, 134, 139, 247 | 메모리 부족 | `--max_memory` 증가 |
| 1 | 일반 오류 | `.nextflow.log`에서 상세 내용 확인 |

대부분의 파이프라인은 실패 전에 2배, 3배 리소스로 자동 재시도합니다.

## HPC/Singularity 문제

### Singularity 캐시 문제
```bash
export NXF_SINGULARITY_CACHEDIR="$HOME/.singularity/cache"
mkdir -p $NXF_SINGULARITY_CACHEDIR
```

### Docker 대신 Singularity 사용
Docker가 없는 HPC 시스템에서는 Singularity를 사용하십시오:
```bash
nextflow run nf-core/<pipeline> -profile singularity ...
```

> **참고**: 기본 환경 설정(Docker, Nextflow, Java 설치)은 SKILL.md 1단계의 인라인 지침을 참조하십시오.

## 파이프라인 실패

### 컨테이너 풀 실패
- 네트워크 연결 상태를 확인하십시오
- docker 대신 `-profile singularity`를 시도하십시오
- 오프라인용: `nf-core download <pipeline> -r <version>`

### "No such file" 오류
- 샘플시트에 **절대 경로**를 사용하십시오
- 파일 존재 확인: `ls /path/to/file`

### Resume가 작동하지 않음
```bash
# Check work directory exists
ls -la work/

# Force clean restart (loses cache)
rm -rf work/ .nextflow*
nextflow run nf-core/<pipeline> ...
```

## RNA-seq 관련

### STAR 인덱스 실패
- 메모리 증가: `--max_memory '64.GB'`
- 또는 사전 빌드된 인덱스 제공: `--star_index /path/to/star/`

### 낮은 정렬률
- 게놈이 종(species)과 일치하는지 확인하십시오
- FastQC에서 어댑터 오염을 확인하십시오
- 다른 정렬기(aligner)를 시도하십시오: `--aligner hisat2`

### 가닥 특이성(strandedness) 감지 실패
- 명시적으로 지정하십시오: `--strandedness reverse`
- 일반적인 값: `forward`, `reverse`, `unstranded`

## Sarek 관련

### BQSR 실패
- 게놈에 사용 가능한 알려진 부위(known sites)를 확인하십시오
- 비표준 참조의 경우 건너뛰기: `--skip_bqsr`

### Mutect2에서 변이 없음
- 종양/정상 페어링을 확인하십시오
- 샘플시트 `status` 열 확인: 0=정상, 1=종양

### WGS 메모리 부족
```bash
--max_memory '128.GB' --max_cpus 16
```

### DeepVariant GPU 문제
- NVIDIA Docker 런타임이 구성되어 있는지 확인하십시오
- 또는 CPU 모드 사용 (더 느림)

## ATAC-seq 관련

### 낮은 FRiP 점수
- `plotFingerprint/`에서 라이브러리 복잡도를 확인하십시오
- 과도한 전위(over-transposition)를 나타낼 수 있습니다

### 피크 수 부족
- 임계값 낮추기: `--macs_qvalue 0.1`
- 브로드 피크 사용: `--narrow_peak false`

### 높은 중복률
- 저입력 시료에서는 정상입니다
- 파이프라인이 기본적으로 제거합니다
- 더 깊은 시퀀싱을 고려하십시오

## 리소스 관리

### 리소스 제한 설정
```bash
--max_cpus 8 --max_memory '32.GB' --max_time '24.h'
```

### 사용 가능한 리소스 확인
```bash
# CPUs
nproc

# Memory
free -h

# Disk
df -h .
```
